# 課題 #003：取引先ランクに基づく商談金額のバリデーション

## 1. 課題の背景と要件
- **背景**: リスク管理のため、取引先の「信頼度ランク」に応じて、一度に契約できる商談金額に上限を設けたい。
- **事前準備**:
    - `Account` オブジェクトにカスタム項目 `Rank__c`（選択リスト: `Gold`, `Silver`, `Bronze`）を作成してください。
- **要件**:
    - 商談の作成・更新時に動作すること（`before insert`, `before update`）。
    - 取引先の `Rank__c` に応じて、以下の金額上限チェックを行う。
        - **Bronze**: 500,000円 まで
        - **Silver**: 5,000,000円 まで
        - **Gold**: 上限なし
        - **ランク未設定**: 100,000円 まで
    - 上限を超えた場合、`Amount` 項目に対して「[ランク名]ランクの取引先は、¥[上限額] 以上の商談を作成できません。」というエラーを表示し、保存を阻止する。
- **技術的ポイント**:
    - `addError` メソッドを使用した保存の阻止。
    - トリガー内で親レコード（Account）の情報を効率的に取得する方法（Mapの活用）。

## 2. Git 運用
- **ブランチ名**: `feature/003-opportunity-validation`
- **コミット名**: `[#003] feat: 取引先ランクに基づく商談金額バリデーションの実装`

## 3. 設計メモ
- **トリガー**: `before insert`, `before update` を追加。
- **ハンドラ**: `checkAmountLimit(List<Opportunity> opps)` メソッドを作成。
    - **注意**: `insert` 時はIDがないため `Map` ではなく `List` を引数にする。
- **処理フロー**:
    1. 引数の商談リストから `AccountId` を `Set` に収集する。
    2. `Account` をSOQLで取得し、`Map<Id, Account>` (ID => Rank__c) を作成する。
    3. 商談リストをループし、`Account` マップからランクを取得して上限チェックを行う。
    4. 超過時は `opp.Amount.addError(...)` でエラーを表示する。
